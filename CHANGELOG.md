# Changelog

This project follows a simplified semantic versioning pattern (e.g. 5.7.4)
1. **MAJOR** – major app changes, new modules
2. **MINOR** – new features, major improvements
3. **PATCH** – fixes, patches, internal refactors
IMPORTANT: new features must be **visible in app** (if not – it's refactors)
